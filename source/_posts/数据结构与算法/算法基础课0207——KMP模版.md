# kmp

**主要思想**： 在字符串匹配问题中，为了提高匹配效率（~~暴力匹配~~），在暴力匹配的基础上改进的算法。在失去匹配的时候，通过一定的技巧，找到能会退的最大长度，而不是无脑的再次直接从0开始匹配。而这个技巧就是kmp中所说的next数组，这个数组的意思是，'next[i]' 表示从0开始的前缀和以i结尾的后缀，相同的字符串的长度。即前缀和后缀相等的最长长度。

# 题目一 模版

数组下标从1开始写，主要是不用考虑一些边界问题。


> 更新时间 2021-12-16
>
> next数组在匹配失败的时候，发挥他的作用，找到回退的最短距离。即找到以失配字符前一个开始 和 模板串开头开始 最长的相等的部分。就是next[j]的值，即匹配是可以跳过重复匹配的长度，从而达到比暴力更优的效果

#include <iostream>

using namespace std;

const int N = 1e5+10, M = 1e6+10;

char p[N],s[M];
int ne[N];
int n,m;

//寻找前缀和后缀相等的最长长度
void getNext(char *p){
  for(int i=2,j=0;i<=n;i++){
    //如果一直不匹配 则 ne[i]直接退化到0
    while(j&&p[i]!=p[j+1])j=ne[j];
    //如果匹配则j++ 相当于最长前缀==最长后缀的 长度加一
    if(p[i]==p[j+1]) j++;
    ne[i]=j;
  }
}

int main(){
  cin>>n>>p+1>>m>>s+1;
  
  getNext(p);
  
  for(int i=1,j=0;i<=m;i++)
  {
    while(j&&s[i]!=p[j+1])j = ne[j];
    if(s[i]==p[j+1])j++;
    if(j==n){
      printf("%d ",i-n);
    }
  }
  return 0;
}
